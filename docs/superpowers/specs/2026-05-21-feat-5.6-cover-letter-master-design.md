## 1. 배경 / 목표

§6 자소서 작성 (회사별 변형본 생성) 의 베이스가 되는 **마스터 자소서** 도메인. 사용자가 항목 유형별로 자기 답변을 미리 작성·저장 → AI 가 회사 지원 시 변형본의 베이스로 활용. v1 은 마스터 CRUD 만 (회사별 변형본·휴지통은 v1 비범위).

## 2. 결정 내역 (브레인스토밍 합의)

1. **암호화**: `master_answer` 컬럼 → 애플리케이션 레이어 AES-256. Spring Security 의 `Encryptors` API 활용 (별도 의존성 X). AES key 는 환경변수.
2. **v1 스코프**: 마스터 탭만. 회사별 변형본·휴지통은 v1 비범위 (UI 의 탭 strip 은 "⭐ 마스터" 단일 탭으로 시작, v2 확장 자리만 마련).
3. **Sub-nav**: 5.5 의 `SidenavRail` 재사용. 카테고리 "이직 / 취업", 항목 2개:
   - "자소서" → `/cover-letters/master`
   - "경력기술서" → `/me/career` (5.4 cross-category link)
4. **TopNav**: "이직/취업" 링크 `/cover-letter` (stub) → `/cover-letters/master` (복수, 정식 경로)

### 데이터 흐름

```
페이지 진입:
  FE  GET /api/v1/cover-letters/masters/summary
   → BE 가 user 의 마스터들을 item_type 별 그룹화 + 카운트
   → FE 좌측 ItemTypeList 채움

항목 유형 선택:
  FE  GET /api/v1/cover-letters/masters?itemType=MOTIVATION
   → BE 가 해당 item_type 의 모든 마스터 (default 우선) 반환
   → BE 가 master_answer 복호화 후 응답
   → FE: default 1개 → MasterEditor, 나머지 → OtherMastersList

저장:
  FE  POST /api/v1/cover-letters/masters { itemType, title?, charLimitHint?, masterAnswer, isDefault }
       또는
  FE  PATCH /api/v1/cover-letters/masters/{id} { title?, charLimitHint?, masterAnswer?, isDefault? }
   → BE 가 masterAnswer 암호화 후 저장
   → isDefault=true 면 같은 (userId, itemType) 의 다른 default 자동 false (트랜잭션)

기본 설정 토글:
  FE  PATCH /api/v1/cover-letters/masters/{id} { isDefault: true }
   → BE: 트랜잭션 안에서 기존 default false 처리 + 본 row default true

복사:
  FE  POST /api/v1/cover-letters/masters/{id}/copy
   → BE: 같은 user/itemType 으로 새 row 생성, title="<원본> 사본", isDefault=false
   → FE 리스트 갱신

삭제:
  FE  DELETE /api/v1/cover-letters/masters/{id}
   → BE hard delete
   → FE: 삭제된 row 가 default 였으면 동일 item_type 의 다른 row 가 default 로 자동 승격
   → BE 응답에 "newDefaultId" 포함 (서버 트랜잭션 안에서 결정)
```

## 6. Backend 상세

### 6.1 Entity

`com.twochi.coverletter.domain.CoverLetterMaster`:

```java
@Entity
@Table(name = "cover_letter_master")
@Getter
@NoArgsConstructor(access = AccessLevel.PROTECTED)
public class CoverLetterMaster {

    public enum ItemType {
        MOTIVATION, FUTURE_PLAN, TEAMWORK, CONFLICT,
        ACHIEVEMENT, PROBLEM_SOLVING, STRENGTH, WEAKNESS, OTHER
    }

    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "user_id", nullable = false)
    private Long userId;

    @Enumerated(EnumType.STRING)
    @Column(name = "item_type", nullable = false, length = 50)
    private ItemType itemType;

    @Column(length = 200)
    private String title;

    @Convert(converter = MasterAnswerConverter.class)  // ← 암호화/복호화 자동
    @Column(name = "master_answer", nullable = false, columnDefinition = "text")
    private String masterAnswer;

    @Column(name = "char_limit_hint")
    private Integer charLimitHint;

    @Column(name = "is_default", nullable = false)
    private boolean isDefault;

    @Column(name = "created_at", nullable = false, updatable = false)
    private Instant createdAt;

    @Column(name = "updated_at", nullable = false)
    private Instant updatedAt;

    // factory methods: create() / update() / setDefault() / unsetDefault()
}
```

### 6.2 MasterAnswerConverter (AttributeConverter)

`com.twochi.coverletter.service.MasterAnswerConverter`:

```java
@Component
@Converter
public class MasterAnswerConverter implements AttributeConverter<String, String> {
    @Autowired
    private MasterAnswerEncryptor encryptor;  // Spring 주입 (정적 lookup 패턴)

    @Override
    public String convertToDatabaseColumn(String plain) {
        return plain == null ? null : encryptor.encrypt(plain);
    }
    @Override
    public String convertToEntityAttribute(String encrypted) {
        return encrypted == null ? null : encryptor.decrypt(encrypted);
    }
}
```

(`@Converter` + `@Autowired` 조합은 JPA spec 외 — Hibernate 가 지원. 또는 `AttributeConverter` Bean 으로 등록 후 entity 의 `@Convert` 가 lookup.)

### 6.3 MasterAnswerEncryptor

`com.twochi.coverletter.service.MasterAnswerEncryptor`:

```java
@Component
public class MasterAnswerEncryptor {

    private final TextEncryptor encryptor;

    public MasterAnswerEncryptor(
        @Value("${master-answer.encryption-key}") String key,
        @Value("${master-answer.encryption-salt}") String salt
    ) {
        if (key == null || key.isBlank()) {
            throw new IllegalStateException(
                "MASTER_ANSWER_ENCRYPTION_KEY 환경변수가 설정되지 않았어요. .env 에 추가하세요."
            );
        }
        // Spring Security: Encryptors.delux() 는 AES/CBC/PKCS5Padding + random IV
        // Encryptors.queryableText() 는 동일 input 에 동일 output (검색 가능, 보안 ↓)
        // 본 케이스는 검색 안 하므로 delux 사용 — 같은 평문도 매번 다른 ciphertext.
        this.encryptor = Encryptors.delux(key, salt);
    }

    public String encrypt(String plain) { return encryptor.encrypt(plain); }
    public String decrypt(String encrypted) { return encryptor.decrypt(encrypted); }
}
```

**환경변수** (신규 2개):
- `MASTER_ANSWER_ENCRYPTION_KEY` — AES 키 (hex 또는 base64). 운영 환경별 다름. 분실 시 기존 데이터 복호화 불가.
- `MASTER_ANSWER_ENCRYPTION_SALT` — hex salt (예: openssl 로 생성)

`.env` 에 추가 (gitignored). `.env.example` 에 placeholder 추가.

### 6.5 Default 승계 로직

`CoverLetterMasterService.delete()`:
```java
public DeleteResult delete(Long userId, Long id) {
    CoverLetterMaster m = findOwned(userId, id);
    boolean wasDefault = m.isDefault();
    ItemType itemType = m.getItemType();
    repository.delete(m);
    if (!wasDefault) return new DeleteResult(null);
    // 남은 마스터 중 가장 최근 수정된 것을 default 로 승격
    return repository.findFirstByUserIdAndItemTypeOrderByUpdatedAtDesc(userId, itemType)
        .map(next -> {
            next.setDefault();
            return new DeleteResult(next.getId());
        })
        .orElse(new DeleteResult(null));
}
```

`setDefault(true)` 트랜잭션 (PATCH):
```java
public void setAsDefault(Long userId, Long id) {
    CoverLetterMaster target = findOwned(userId, id);
    repository.findByUserIdAndItemTypeAndIsDefaultTrue(userId, target.getItemType())
        .ifPresent(existing -> existing.unsetDefault());
    target.setDefault();
}
```

## 7. 검증 / 에러

- 클라이언트:
  - `masterAnswer` 빈 값 → "답변을 입력해주세요"
  - `title` 길이 > 200 → "제목은 200자 이내로 작성해주세요"
  - `charLimitHint` 음수 → "글자수는 0보다 커야 해요"
- 서버:
  - validation 어노테이션 + `BusinessException(ErrorCode.MASTER_NOT_FOUND)` (신규)
  - 암호화 실패 → 500 INTERNAL_ERROR (로그 + 키 설정 확인 메시지)

## 11. 리스크 / 미해결

- **암호화 키 분실 시 모든 마스터 답변 복호화 불가**. 운영에서 키 백업 필수. v1 베타는 단일 사용자라 위험 낮으나 절차 문서화 필요.
- **암호화 컬럼 검색 불가** — `master_answer` 의 LIKE/full-text 검색 v1 불필요 (시안에 검색 UI 없음). 추후 검색 도입 시 별도 검색 전용 컬럼 (queryableText) 또는 메타데이터만 검색.
- `is_default=TRUE` unique partial index — 두 default 동시 INSERT 레이스 가능성. 트랜잭션 isolation REPEATABLE_READ 또는 SELECT FOR UPDATE 로 대응 (대용량 아니면 default isolation 으로 충분).
- 마스터 답변 길이 — schema 는 TEXT (무제한), v1 클라이언트 검증으로 ~5000자 권장.
- §6 자소서 작성 (회사별 변형본) 에서 본 마스터를 어떻게 참조할지 (`master_id` FK) — schema 에 이미 정의됨. 본 spec 은 마스터 도메인만, §6 spec 이 별도.
