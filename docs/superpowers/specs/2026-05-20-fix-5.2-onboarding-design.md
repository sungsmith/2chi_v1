### 2.2 Step 2 — Career (`careerYear: number`)

시안 8개 chip 그대로 표시. UI 값 ↔ 데이터:

| 시안 chip | UI 라벨 | 저장 값 (number) |
|---|---|---|
| 신입 | "신입 / 0년" | `0` |
| 1년차 | "1 년차" | `1` |
| 2~6년차 | "N 년차" | `2`~`6` |
| 7+ 년차 이상 | "7+ 년차 이상" | `7` (sentinel) |

→ 기존 5.2 의 `careerYear: number` 그대로. BE OnboardingRequest 와 호환.

### 2.3 Step 3 — Positions (`TargetJob` 5개)

| 시안 POSITIONS | 매핑 enum | 시안 tone | desc 처리 |
|---|---|---|---|
| `fe` (Frontend, 웹·앱 화면 구현) | `TargetJob.FRONTEND` | tone-1 | 시안 desc 유지 ("웹·앱 화면 구현") |

| `infra` (Infra / Cloud, AWS · GCP · K8s) | `TargetJob.INFRA_CLOUD` | tone-3 | 시안 desc 유지 |

→ 시안 5개로 줄어든 `.position-grid` 의 시각 정리는 CSS 변환 단계에서 조정 (8개 그리드를 5개로 자연스럽게 채우는 룰).

### 2.4 Step 4 — Recap (선택 데이터 표시)

```
recap-row "준비":  TARGET_META[target].title              ex. "이직 준비 중"
recap-row "경력":  YEARS[careerYear].num + lbl            ex. "3 년차" | "신입 0년" | "7+ 년차 이상"

                                                          ex. "Backend · Infra / Cloud"
```

Sample-grid 3개 카드는 시안 SAMPLES 정적 (PRAR / 자소서 / 공고별).

### 4.2 Stepper

```tsx
const STEPS = [
  { num: 1, label: "준비 단계" },
  { num: 2, label: "경력 연차" },
  { num: 3, label: "희망 직무" },
  { num: 4, label: "둘러보기" },
];

export function Stepper({ current }: { current: 1 | 2 | 3 | 4 }) {
  return (
    <div className="onb-stepper">
      {STEPS.map((s, i) => {
        const idx = i + 1;
        const state = idx < current ? "done" : idx === current ? "active" : "";
        return (
          <div key={s.num} className={`onb-step-pill ${state}`}>
            <div className="bar" />
            <div className="meta">
              <span className="num">{idx < current ? <Check size={10} /> : s.num}</span>
              <span>{s.label}</span>
            </div>
          </div>
        );
      })}
    </div>
  );
}
```

### 현재 (5.2)

```

  → setSubmitting(true)
```

### 6.1 복사 시 변환 (사전 분석으로 라인 단위 확정)

5.3-dashboard 의 dashboard.css 변환과 동일 패턴. 시안 onboarding.css 사전 분석 결과:

| 항목 | 시안 (raw line) | 변환 결과 |
|---|---|---|
| `body.onb-body { ... }` (L5–13) | `margin/background/min-height/font-family/color/overflow-x/position: relative` | → `.onb-stage { position: relative; overflow-x: hidden; }` 로 축소 (`margin/background/font-family/color` 는 globals.css 가 이미 담당. `min-height` 는 `<main>` 이 담당하지만 (public)/layout 에는 `<main>` wrapper 없음 → `.onb-stage` 자체에 `min-height: 100vh` 유지) |
| `body.onb-body::before { ... }` (L18-35) | ambient blob A (position: fixed, lavender + primary gradient + 28s 애니메이션) | → `.onb-stage::before { ... }` 그대로. position: fixed 유지 (5.3 의 fixed 사용 패턴과 동일, 5.3 review 의 risk 주석 동일하게 추가) |
| `body.onb-body::after { ... }` (L36–45) | ambient blob B (position: fixed, mint + peach + pink) | → `.onb-stage::after { ... }` 그대로 |
| `@keyframes onbBlobA/B` | body 스코프 아님 | 그대로 유지 |
| `@import url(...)` / `@font-face` | **없음** (확인됨) | 작업 불필요 |
| `.doc-*` / `.wf-*` / `.dash-nav-*` chrome 클래스 | **없음** (확인됨) | 작업 불필요 |

**`min-height: 100vh` 결정 보완**:
- (public)/layout.tsx 가 `<main>` wrapper 없이 `{children}` 직접 렌더 (인증 가드 페이지들이 자체 `<main>` 보유)
- `(public)/onboarding/page.tsx` 의 `<main className="onb-stage">` 가 풀 viewport 차지해야 함 → `.onb-stage` 에 `min-height: 100vh` 보존
- 5.3 의 `min-height: calc(100vh - 64px)` 패턴과 다름 (5.3 은 (app)/layout 의 `<main>` 이 이미 calc 적용)

**ambient blob 주석 추가**:
시안의 `body.onb-body::before/::after` 가 `position: fixed` + 28s/34s 애니메이션 (will-change: transform) 사용. 5.3 review 의 risk 주석과 동일 패턴 적용:

```css
/* position: fixed intentional — creates viewport-attached ambient wash.
   IMPORTANT: any transform/filter on an ancestor (.onb-stage or its parents)
   will confine these to that element. Verify after Task 1 visual check. */
```

### 6.2 페이지 통합

`(public)/onboarding/page.tsx`:

```tsx
"use client";
import "./onboarding.css";
import { useEffect } from "react";
import { useRouter } from "next/navigation";
import { useAuth } from "@/contexts/auth-context";
import { OnboardingFlow } from "@/components/onboarding/onboarding-flow";

export default function OnboardingPage() {
  const { user, initialized } = useAuth();
  const router = useRouter();

  useEffect(() => {
    if (!initialized) return;
    if (!user) { router.replace("/login"); return; }
    if (user.onboardingCompleted) router.replace("/");
  }, [initialized, user, router]);

  if (!initialized || !user || user.onboardingCompleted) return null;

  return (
    <main className="onb-stage">
      <OnboardingFlow />
    </main>
  );
}
```

(현 page.tsx 와 차이: `<main style={{ maxWidth: 720, ... }}>` → `<main className="onb-stage">`, import 추가)
