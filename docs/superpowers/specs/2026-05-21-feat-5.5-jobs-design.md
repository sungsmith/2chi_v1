## 1. 배경 / 목표

5.6 자소서 작성·5.7 캘린더·5.8 지원 현황 의 데이터 소스가 되는 **채용공고** 도메인을 v1 첫 화면으로 구축. 시안 §4 의 v1 스코프: URL 붙여넣기 자동 파싱 + 직접 작성 + 최근 등록 리스트. 사람인 OpenAPI 검색은 v2 (탭 표시만, disabled).

## 2. 결정 내역 (브레인스토밍 합의)

1. **URL 파싱 정책**: 사람인 (`saramin.co.kr`) + 잡코리아 (`jobkorea.co.kr`) 2개 사이트만 BE 가 Jsoup 으로 fetch + parse. 실패 시 직접 작성 탭으로 자동 폴백 (사용자 클릭 X).
2. **JD 키워드 추출**: 저장 시 BE → **Anthropic Claude API** 호출 → `keywords: string[]` 추출. 비용 공고당 1회.
3. **Sub-nav**: 좌측 240px sidenav rail (`기업` 카테고리, 항목: 채용공고·기업분석). §9 도 동일 rail 공유.
4. **v1 스코프**: 단일 페이지 `/company/postings` 에 (상단) 등록 폼 + (하단) "내가 등록한 공고" 카드 리스트. 항목 클릭 → detail/edit 모달. 수정·삭제 가능.
5. **TopNav "기업" 링크 변경**: `/jobs` → `/company/postings`.

### 데이터 흐름

```
URL 파싱:
  FE  POST /api/postings/parse { url } 
   → BE PostingParseController → PostingParserService 
   → SaraminParser/JobkoreaParser.parse() 
   → ParsedPosting { company, title, jobRole, deadline, mainTasks, requirements, preferred, sourceUrl }
   → FE 폼 필드에 자동 채움 (사용자가 검토·수정 후 저장)
  실패: BE 403/422 응답 → FE 가 직접 작성 탭으로 자동 전환 + 알림 banner

저장:
  FE  POST /api/postings { source, company, title, ..., sourceUrl? }
   → BE JobPostingService.create()
   → ClaudeKeywordExtractor.extract(mainTasks + requirements + preferred) 
   → 동기 응답 (keywords 포함)
   → FE 리스트 prepend + 폼 reset
  
리스트:
  FE  GET /api/postings → JobPostingResponse[]
  
수정/삭제:
  FE  PATCH /api/postings/{id} → 응답으로 새 키워드 (텍스트 변경 시 재추출)
  FE  DELETE /api/postings/{id}
```

## 6. Backend 상세

### 6.1 Entity 매핑 (기존 V1 schema 그대로)

`com.twochi.posting.JobPosting`:
```java
@Entity
@Table(name = "job_posting")
public class JobPosting {
  @Id @GeneratedValue Long id;
  @Column(nullable = false) Long userId;
  @Enumerated(EnumType.STRING) @Column(nullable = false) Source source;
  @Column(nullable = false, length = 200) String company;
  @Column(nullable = false, length = 300) String title;
  String jobRole;
  @Column(columnDefinition = "text") String requirements;
  @Column(columnDefinition = "text") String preferred;
  @Column(columnDefinition = "text") String mainTasks;
  LocalDate deadline;
  String sourceUrl;
  @Type(StringArrayType.class) @Column(columnDefinition = "text[]") List<String> keywords = new ArrayList<>();
  Instant createdAt; Instant updatedAt;
  enum Source { URL, MANUAL, SARAMIN }
}
```

`text[]` 매핑은 hibernate-types-60 라이브러리 (BE 의존성 추가 필요) 또는 `@Convert` 커스텀 컨버터. 기존 BE 패턴 따름.

### 6.2 API

| 메서드 | 경로 | 설명 |
|---|---|---|
| POST | `/api/postings/parse` | `{ url }` → `ParsedPosting` 또는 422 |
| GET | `/api/postings` | 현재 사용자 공고 리스트 (created_at desc) |
| GET | `/api/postings/{id}` | 단건 조회 |
| POST | `/api/postings` | `JobPostingCreateRequest` → 저장 + keywords 추출 → `JobPostingResponse` |
| PATCH | `/api/postings/{id}` | 부분 수정. mainTasks/requirements/preferred 중 하나라도 변경 시 keywords 재추출 |
| DELETE | `/api/postings/{id}` | 소프트 삭제 X, 하드 삭제 |

모든 endpoint 는 `@AuthenticatedUserId` (5.1 패턴) 로 user 격리.

### 6.3 URL Parser

`com.twochi.posting.parser.PostingParser`:
```java
public interface PostingParser {
  boolean supports(URI url);
  ParsedPosting parse(URI url) throws ParsingException;
}
```

구현 2개:
- `SaraminParser`: `saramin.co.kr` 매칭. Jsoup `Jsoup.connect(url).userAgent(UA).get()` + CSS selector (e.g. `.wrap_jv_cont .cont` 등 사이트 구조 분석 후 결정).
- `JobkoreaParser`: `jobkorea.co.kr` 동일 패턴.

`PostingParserService`:
```java
public ParsedPosting parse(URI url) {
  PostingParser parser = parsers.stream()
    .filter(p -> p.supports(url))
    .findFirst()
    .orElseThrow(() -> new UnsupportedSiteException(url));
  return parser.parse(url);
}
```

`ParsingException`/`UnsupportedSiteException` → ControllerAdvice 가 422 (body: `{ "message": "이 사이트는 자동 파싱을 지원하지 않아요. 직접 작성으로 입력해주세요." }`).

신규 BE 의존성:
- `org.jsoup:jsoup:1.18.x` (최신 안정 버전)

### 6.4 Keyword Extractor (Anthropic API)

`com.twochi.posting.keyword.ClaudeKeywordExtractor`:
- env var `ANTHROPIC_API_KEY` 필수. 없으면 startup 실패 (fail-fast).
- 모델: `claude-haiku-4-5-20251001` (저렴·빠름, 키워드 추출은 단순 작업)
- Prompt:
  ```
  다음 채용공고에서 핵심 기술·키워드를 10개 이내로 추출하세요.
  결과는 JSON 배열만 반환. 다른 텍스트 없이.
  예: ["Spring Boot", "MSA", "Kafka"]
  
  ---
  주요 업무: {mainTasks}
  자격 요건: {requirements}
  우대 사항: {preferred}
  ```
- HTTP 호출: Spring 의 `RestClient` (Java 21 / Spring Boot 3.x 의 권장)
- 응답 파싱 실패 / API 호출 실패 → 빈 배열 반환 (저장은 진행). 로그 WARN.
- 호출 비용 ~$0.001/공고 (Haiku 4.5 기준)

신규 BE 의존성: 없음 (RestClient 는 Spring 기본 포함)

신규 환경변수: `ANTHROPIC_API_KEY`

### 7.1 URL 파싱 실패 시 자동 폴백

## 9. 리스크 / 미해결

- 사람인·잡코리아 의 HTML 구조 변경 시 파서 깨짐 → 단위 테스트 fixture 가 늦게 알려줌. 운영 모니터링 별도 필요 (v2 후속).
- Anthropic API 응답 latency 0.5~2초 → 저장 응답 시간 증가. FE 는 저장 버튼 disabled + spinner 로 처리.
- LLM 응답 파싱 실패 시 빈 배열로 graceful — 키워드 없는 공고가 5.6 자소서 매칭 품질 영향 가능. fallback policy: 빈 배열이면 5.6 에서 기본 키워드 추출 재시도 (5.6 spec 의 책임).
- 사용자가 사람인 비회원 페이지 URL 만 입력 시 fetch 가능 — 회원 전용 페이지 파싱 시 빈 결과 (graceful failure 로 처리).
