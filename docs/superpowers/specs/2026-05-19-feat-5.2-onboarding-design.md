## 1. 범위

**포함:**
- `POST /api/v1/onboarding` — Profile 생성/갱신 (멱등), onboarding_completed=true
- `GET /api/v1/users/me` 응답 확장 — `onboardingCompleted` 필드
- 프론트 `/onboarding` 4단계 페이지 (디자인 시스템 onboarding.jsx 의도 포팅)
- AuthContext 확장 — onboardingCompleted 보유, 로그인 후 라우팅 분기
- 홈 (`/`) 미완료 사용자 배너 + 온보딩 CTA
- 백엔드 Profile JPA 매핑 (V1 스키마에 이미 테이블 존재)

**옵션 (스펙 v1 기준 축소):**
- PURPOSES 2개: `JOB_CHANGE` / `EMPLOYMENT`
- CAREERS 8개: 0~7+ (디자인 그대로)
- POSITIONS 5개: `BACKEND` / `FRONTEND` / `INFRA_CLOUD` / `INFRA_OPS` / `UI_UX`

**제외 (다음 브랜치 / v2):**
- 다른 PURPOSES (포트폴리오/탐색) — v2 확장
- 다른 POSITIONS (Publisher/Data/기타) — v2
- 내 정보(§5.4) — name/birth/phone 등 Profile 의 나머지 필드
- 인프라 트랙 자동 판정 — v1 사용자 직접 선택
- Step 4 의 "샘플 자소서 1건 자동 생성" 데모 — 디자인이 이미 recap 으로 단순화. AI 의존 없음.

## 2. 전체 흐름

```
[가입 직후]
  /signup → 201 → /login (기존)

[첫 로그인]
  /login → 200 → AuthContext user.onboardingCompleted
                  ├─ false → router.push('/onboarding')
                  └─ true  → router.push('/')

[온보딩]
  /onboarding (로그인 필요)
    Step 1: 접속 목적 (JOB_CHANGE | EMPLOYMENT) — 단일 선택
    Step 2: 경력 (0~7+) — 단일 선택
    Step 3: 희망 직무 (multi, 1개 이상) — 5개 중
    Step 4: Recap 요약
       └─ [완료] → POST /api/v1/onboarding → user 갱신 → router.push('/')
       └─ [나중에] → router.push('/') (Profile 미생성)

[홈]
  / → AuthContext user.onboardingCompleted = false 면 배너
       "온보딩을 완료하면 맞춤 분석이 시작돼요 → [지금 완료하기]"
```

### 3.3 Profile Entity

```java
@Entity
@Table(name = "profile")
@Getter
@NoArgsConstructor(access = AccessLevel.PROTECTED)
public class Profile {

    @Id
    @Column(name = "user_id")
    private Long userId;

    @Enumerated(EnumType.STRING)
    @Column(nullable = false, length = 20)
    private Target target;

    @Column(name = "career_year", nullable = false)
    private short careerYear;

    // PostgreSQL text[] 매핑: String[] 으로 저장하고 enum 변환은 서비스 단에서 처리
    @Column(name = "target_jobs", nullable = false, columnDefinition = "text[]")
    @JdbcTypeCode(SqlTypes.ARRAY)
    private String[] targetJobsRaw;

    @Column(name = "onboarding_completed", nullable = false)
    private boolean onboardingCompleted;

    @Column(name = "created_at", nullable = false, updatable = false)
    private Instant createdAt;

    @Column(name = "updated_at", nullable = false)
    private Instant updatedAt;

    // 도메인 메서드
    public List<TargetJob> targetJobs() {
        return Arrays.stream(targetJobsRaw).map(TargetJob::valueOf).toList();
    }

    public static Profile create(Long userId, Target target, short careerYear, Set<TargetJob> jobs, Instant now) {
        Profile p = new Profile();
        p.userId = userId;
        p.target = target;
        p.careerYear = careerYear;
        p.targetJobsRaw = jobs.stream().map(Enum::name).toArray(String[]::new);
        p.onboardingCompleted = true;
        p.createdAt = now;
        p.updatedAt = now;
        return p;
    }

    public void updateFromOnboarding(Target target, short careerYear, Set<TargetJob> jobs, Instant now) {
        this.target = target;
        this.careerYear = careerYear;
        this.targetJobsRaw = jobs.stream().map(Enum::name).toArray(String[]::new);
        this.onboardingCompleted = true;
        this.updatedAt = now;
    }
}
```

### 3.4 API 계약

**POST `/api/v1/onboarding`** (인증 필요)

Request:
```json
{
  "target": "JOB_CHANGE",
  "careerYear": 3,
  "targetJobs": ["BACKEND", "INFRA_CLOUD"]
}
```

Validation:
| 필드 | 규칙 |
|---|---|
| target | `@NotNull` Target enum |
| careerYear | `@NotNull`, `@Min(0)`, `@Max(7)` (7+ 은 7로 저장) |
| targetJobs | `@NotEmpty Set<TargetJob>` |

Response 200:
```json
{
  "userId": 1,
  "target": "JOB_CHANGE",
  "careerYear": 3,
  "targetJobs": ["BACKEND", "INFRA_CLOUD"],
  "onboardingCompleted": true
}
```

멱등: 동일 user 가 두 번 호출 → UPDATE. 응답은 항상 200 (201 분기 없음).

Errors:
| HTTP | code | 상황 |
|---|---|---|
| 400 | VALIDATION_FAILED | enum invalid / careerYear 범위 / targetJobs 빈 배열 / null |
| 401 | UNAUTHENTICATED | Access 토큰 없음 |

### 3.5 OnboardingService

```java
@Service
public class OnboardingService {
    private final ProfileRepository profileRepository;

    @Transactional
    public OnboardingResponse upsert(Long userId, OnboardingRequest req) {
        Instant now = Instant.now();
        short year = req.careerYear().shortValue();
        Profile profile = profileRepository.findById(userId)
            .map(existing -> { existing.updateFromOnboarding(req.target(), year, req.targetJobs(), now); return existing; })
            .orElseGet(() -> profileRepository.save(Profile.create(userId, req.target(), year, req.targetJobs(), now)));

        return new OnboardingResponse(
            userId, profile.getTarget(), profile.getCareerYear(),
            profile.targetJobs(), profile.isOnboardingCompleted()
        );
    }
}
```

### 4.4 lib/api/onboarding.ts

```ts
import { apiUrl, http } from "./http";
import { Target } from "@/lib/enums/target";
import { TargetJob } from "@/lib/enums/target-job";

export type OnboardingPayload = {
  target: Target;
  careerYear: number;
  targetJobs: TargetJob[];
};

export async function postOnboarding(payload: OnboardingPayload): Promise<void> {
  const res = await http(apiUrl("/api/v1/onboarding"), {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload),
  });
  if (!res.ok) {
    const body = await res.json();
    throw new Error(body.message ?? "온보딩 저장 실패");
  }
}
```

### 4.5 `/onboarding` 페이지 (가드)

```tsx
"use client";

import { useAuth } from "@/contexts/auth-context";
import { OnboardingFlow } from "@/components/onboarding/onboarding-flow";
import { useRouter } from "next/navigation";
import { useEffect } from "react";

export default function OnboardingPage() {
  const { user, initialized } = useAuth();
  const router = useRouter();

  useEffect(() => {
    if (!initialized) return;
    if (!user) { router.replace("/login"); return; }
    if (user.onboardingCompleted) router.replace("/");
  }, [initialized, user, router]);

  if (!initialized || !user || user.onboardingCompleted) return null;

  return <OnboardingFlow />;
}
```

### 4.6 OnboardingFlow

상태: `current (0~3)`, `target`, `careerYear`, `targetJobs (Set)`, `submitting`, `topError`.

- 단계 검증:
  - Step 0: `!!target`
  - Step 1: `careerYear !== undefined`
  - Step 2: `targetJobs.size > 0`
  - Step 3: `true`
- 버튼: [이전] / [다음] / Step 3 에서 [완료] + [나중에]
- [완료] → `postOnboarding({...})` → `refreshUser()` → `router.push('/')`
- [나중에] → `router.push('/')` (Profile 미생성)
- 디자인 토큰 적용 inline style. 감성 요소 미사용.
