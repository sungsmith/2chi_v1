## 2. 전체 흐름

```
사용자 → /signup
  ├── 이메일·비밀번호·닉네임 입력 (실시간 검증)
  ├── 만 14세 이상 체크 (필수)
  ├── 약관·개인정보 동의 (필수) + 마케팅 동의 (선택)
  │     └── /terms, /privacy 링크 (새 탭)
  └── [가입하기]
       │ POST /api/v1/auth/signup
       ▼
백엔드 ┬── 입력 검증 (Bean Validation + 비즈니스)
       ├── 트랜잭션:
       │    1. User 생성 (email_verified=false, bcrypt 해시)
       │    2. ConsentLog: terms, privacy, marketing 각각 1행
       │       (Profile 은 온보딩(5.2)에서 target 선택 후 생성. target 컬럼이 NOT NULL+CHECK)
       └── 201 Created
       │
       ▼
프론트 "가입 완료" 안내 → router.push('/login')
       │
       ▼
/login (placeholder. 실제 로그인은 다음 브랜치)
```

### 3.2 API 계약

**`POST /api/v1/auth/signup`**

Request:
```json
{
  "email": "user@example.com",
  "password": "Pass1234!",
  "nickname": "홍길동",
  "ageConfirmed": true,
  "consents": {
    "terms": true,
    "privacy": true,
    "marketing": false
  }
}
```

Validation:
| 항목 | 규칙 |
|---|---|
| email | RFC 5322, 255자 이하, DB unique (alive 사용자) |
| password | 8자+, 영문/숫자/특수문자 중 2종 이상 |
| nickname | 2~20자, 한·영·숫자만, DB unique (alive 사용자) |
| ageConfirmed | true 필수 |
| consents.terms / privacy | true 필수 |
| consents.marketing | true/false 무관 |

Response — **201 Created**:
```json
{ "userId": 123, "email": "user@example.com", "nickname": "홍길동" }
```

Errors (글로벌 핸들러 `{ code, message, traceId }`):
| HTTP | code | 상황 |
|---|---|---|
| 400 | VALIDATION_FAILED | Bean Validation 위반 |
| 409 | EMAIL_DUPLICATE | 이미 가입된 이메일 |
| 409 | NICKNAME_DUPLICATE | 이미 사용중인 닉네임 |
| 422 | AGE_NOT_CONFIRMED | ageConfirmed=false |
| 422 | REQUIRED_CONSENT_MISSING | terms 또는 privacy 미동의 |

### 3.3 Spring Security 의존성 + 설정

`build.gradle.kts` 에 추가:
```kotlin
implementation("org.springframework.boot:spring-boot-starter-security")
```

`SecurityConfig`:
- CSRF disable (JWT 도입 시까지 임시)
- session STATELESS
- `/api/v1/auth/**`, `/error` → permitAll
- 그 외 authenticated
- `PasswordEncoder` bean = `BCryptPasswordEncoder(12)`

### 3.4 V1 스키마 확인

V1__init_schema.sql 이 이미 `app_user` 테이블명을 사용 중 (PostgreSQL `user` 예약어 회피). 별도 수정 불필요.

### 4.3 폼 검증 흐름

실시간 검증:
- 이메일: RFC 5322 정규식
- 비밀번호: 8자+ 2종 조합
- 닉네임: 2~20자, `[가-힣a-zA-Z0-9]`

[가입하기] 클릭:
1. 모든 필드 검증 통과 → 아니면 첫 에러 필드 포커스
2. `POST /api/v1/auth/signup`
3. 201 → "가입 완료" 안내 → `router.push('/login')`
4. 409 EMAIL_DUPLICATE → 이메일 필드 인라인 에러
5. 409 NICKNAME_DUPLICATE → 닉네임 필드 인라인 에러
6. 4xx 기타 → 상단 알림
7. 5xx → "잠시 후 다시 시도해주세요"
